Q1. What is Java?

‚úì Correct Answer: A high-level, object-oriented, platform-independent programming language

Java is a high-level programming language (easy for humans to read/write), object-oriented (based on objects and classes), and platform-independent (runs on any system with JVM through bytecode).

Q2. Who created Java?

‚úì Correct Answer: James Gosling

James Gosling, along with Mike Sheridan and Patrick Naughton, created Java at Sun Microsystems in the early 1990s. Dennis Ritchie created C, Bjarne Stroustrup created C++, and Guido van Rossum created Python.

Q3. In which year was Java officially released?

‚úì Correct Answer: 1995

Java 1.0 was officially released in 1995. The project started in 1991 as "Project Green" and was initially named "Oak" before being renamed to Java in 1995.

Q4. What was Java originally called before being renamed?

‚úì Correct Answer: Oak

Java was originally named "Oak" after an oak tree outside James Gosling's office. It was renamed to "Java" in 1995 due to trademark issues. Java is named after Java coffee, which the developers loved.

Q5. What does WORA stand for in Java?

‚úì Correct Answer: Write Once, Run Anywhere

WORA (Write Once, Run Anywhere) is Java's core philosophy. You write Java code once, compile it to bytecode, and it runs on any platform that has a JVM without modification or recompilation.

Q6. What does JVM stand for?

‚úì Correct Answer: Java Virtual Machine

JVM stands for Java Virtual Machine. It's an abstract computing machine that provides a runtime environment to execute Java bytecode. The JVM is what makes Java platform-independent.

Q7. What does JRE stand for?

‚úì Correct Answer: Java Runtime Environment

JRE stands for Java Runtime Environment. It includes the JVM plus core libraries needed to run Java applications. You need JRE to run Java programs, but not to develop them.

Q8. What does JDK stand for?

‚úì Correct Answer: Java Development Kit

JDK stands for Java Development Kit. It's a complete software development kit that includes JRE (which includes JVM) plus development tools like javac (compiler), javadoc, jar, and debugger.

Q9. What is the relationship between JDK, JRE, and JVM?

‚úì Correct Answer: JDK contains JRE, which contains JVM

The relationship is hierarchical: JDK ‚äÉ JRE ‚äÉ JVM. JDK is the complete development kit containing JRE (runtime environment) which contains JVM (execution engine). To develop Java apps, you need JDK. To only run them, JRE is sufficient.

Q10. What is the primary purpose of JRE?

‚úì Correct Answer: To run Java applications

JRE (Java Runtime Environment) is designed to run Java applications. It includes the JVM and core libraries but NOT development tools like the compiler. If you only want to run Java programs (not develop them), JRE is what you need.

Q11. What is the file extension of Java bytecode?

‚úì Correct Answer: .class

Java bytecode files have the .class extension. When you compile a .java source file using javac, it creates a .class file containing platform-independent bytecode that the JVM can execute.

Q12. Which command is used to compile Java source code?

‚úì Correct Answer: javac

The javac command (Java Compiler) is used to compile Java source code (.java files) into bytecode (.class files). Example: javac HelloWorld.java creates HelloWorld.class.

Q13. Which command is used to run a compiled Java program?

‚úì Correct Answer: java

The java command launches the JVM and runs the compiled bytecode. Example: java HelloWorld (note: no .class extension needed). The JVM loads the .class file and executes it.

Q14. What is Java bytecode?

‚úì Correct Answer: Platform-independent intermediate code

Java bytecode is platform-independent intermediate code that sits between source code and machine code. It's not specific to any processor or OS. The JVM interprets/compiles this bytecode to native machine code for the specific platform it's running on.

Q15. How does Java achieve platform independence?

‚úì Correct Answer: Bytecode is platform-independent

Java achieves platform independence through bytecode. Source code compiles to bytecode (not native machine code). This bytecode runs on any platform with a JVM. The JVM is platform-specific, but bytecode is not. This is the "Write Once, Run Anywhere" magic!

Q16. Is the JVM platform-dependent?

‚úì Correct Answer: Yes, different JVMs for different platforms

The JVM itself IS platform-dependent! There's a Windows JVM, macOS JVM, Linux JVM, etc. Each is specifically designed for its platform. However, they all execute the SAME bytecode, which is why Java programs are platform-independent.

Q17. Which of the following is a major use case for Java?

‚úì Correct Answer: Android applications

Android applications are primarily written in Java (and Kotlin, which runs on JVM). Over 3 billion Android devices run Java-based applications. Java is also used for enterprise applications, web backends, and big data, but NOT for device drivers or OS kernels (which require low-level languages like C).

Q18. Which popular game is written in Java?

‚úì Correct Answer: Minecraft

Minecraft (Java Edition) is written in Java! This is why it runs on Windows, Mac, and Linux without modification. The game's platform independence is a perfect demonstration of Java's "Write Once, Run Anywhere" capability.

Q19. Where is Java commonly used in the industry?

‚úì Correct Answer: Enterprise applications, web backends, Android apps, big data

Java is incredibly versatile! It's used in: (1) Enterprise applications (banking, insurance), (2) Web backends (Spring Boot), (3) Android apps, (4) Big data (Hadoop, Spark, Kafka), (5) Cloud applications, (6) Scientific applications, and more. 90% of Fortune 500 companies use Java.

Q20. Which big data technologies are written in Java?

‚úì Correct Answer: Apache Hadoop, Apache Spark, Apache Kafka

Major big data frameworks are written in Java: Apache Hadoop (distributed data processing), Apache Spark (fast analytics), Apache Kafka (real-time streaming), and Elasticsearch (search engine). Java's performance and scalability make it ideal for processing massive datasets.

Q21. Why is Java still popular today despite being nearly 30 years old?

‚úì Correct Answer: Rich ecosystem of libraries and frameworks

Java remains popular due to: (1) Rich ecosystem (Spring, Hibernate, etc.), (2) Large community (9+ million developers), (3) Strong job market, (4) Backward compatibility, (5) LTS versions with long-term support, (6) Performance improvements (modern JVM is fast), and (7) Versatility (works everywhere).

Q22. What does LTS stand for in Java versions?

‚úì Correct Answer: Long-Term Support

LTS stands for Long-Term Support. LTS versions (like Java 8, 11, 17, 21) receive extended support, security updates, and bug fixes for many years. Enterprises prefer LTS versions for production systems due to stability and guaranteed support.

Q23. Which of the following are LTS (Long-Term Support) versions of Java?

‚úì Correct Answer: 8, 11, 17, 21

The LTS versions are Java 8 (2014), Java 11 (2018), Java 17 (2021), and Java 21 (2023). These versions receive extended support and are recommended for production use. Non-LTS versions have shorter support periods.

Q24. Which major feature was introduced in Java 8?

‚úì Correct Answer: Lambda expressions and Stream API

Java 8 (2014) introduced lambda expressions and Stream API, bringing functional programming features to Java. This was one of the most significant updates. Generics came in Java 5, module system in Java 9, and virtual threads in Java 21.

Q25. What does JIT stand for in the context of JVM?

‚úì Correct Answer: Just-In-Time compiler

JIT (Just-In-Time) compiler is part of the JVM that compiles frequently executed bytecode into native machine code during runtime. This significantly improves performance by avoiding repeated interpretation of the same code.

Q26. What is the correct signature of the main method in Java?

‚úì Correct Answer: public static void main(String[] args)

The main method must be: public (accessible from anywhere), static (can be called without creating an object), void (doesn't return a value), and named "main" with String[] args parameter. This is the entry point the JVM looks for.

Q27. Why is the main method static in Java?

‚úì Correct Answer: JVM can call it without creating an object

The main method is static because when the JVM starts your program, no objects exist yet! The JVM needs to call a method to start execution, and it can't create an object first. That's why main() must be static - it belongs to the class, not to instances.

Q28. What does it mean that Strings are immutable in Java?

‚úì Correct Answer: Strings are immutable in Java

String immutability means once a String object is created, its content cannot be changed. Methods like concat(), replace(), toUpperCase() don't modify the original string - they return NEW strings. This ensures thread safety and enables string pooling for memory optimization.

Q29. For which scenario is Java NOT the best choice?

‚úì Correct Answer: Ultra-low-level hardware programming

Java is NOT ideal for ultra-low-level hardware programming (device drivers, OS kernels, tiny embedded systems) because the JVM overhead is too heavy. For these use cases, C, C++, or Assembly are better. Java excels at enterprise apps, Android, web backends, and big data.

Q30. What is automatic memory management in Java called?

‚úì Correct Answer: Garbage collection

Garbage collection is Java's automatic memory management system. The JVM automatically identifies and frees memory occupied by objects that are no longer in use. This prevents memory leaks and makes Java programming safer and easier compared to manual memory management in C/C++.

Q31. Approximately how many Java developers are there worldwide?

‚úì Correct Answer: Over 9 million

There are over 9 million Java developers worldwide, making it one of the largest programming communities. This massive community means extensive resources, libraries, frameworks, and support available for Java developers.

Q32. What percentage of Fortune 500 companies use Java?

‚úì Correct Answer: 90%

Approximately 90% of Fortune 500 companies use Java in their technology stack. This demonstrates Java's dominance in enterprise software development. Companies like Amazon, Google, Netflix, LinkedIn, and major banks rely heavily on Java.
