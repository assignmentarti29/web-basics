Program 1: ArrayList Basic Usage

Demonstrates adding, accessing, and removing elements in an ArrayList.

â˜• Java
import java.util.ArrayList;
import java.util.List;

public class ArrayListDemo {
    public static void main(String[] args) {
        // Create an ArrayList
        List fruits = new ArrayList<>();

        // Add elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Orange");
        fruits.add("Apple"); // Duplicates allowed

        System.out.println("Fruits List: " + fruits);

        // Access element by index
        String firstFruit = fruits.get(0);
        System.out.println("First Fruit: " + firstFruit);

        // Remove element
        fruits.remove("Banana");
        System.out.println("After removing Banana: " + fruits);

        // Check size
        System.out.println("Total fruits: " + fruits.size());
    }
}

Baby: Like adding toys to a toy box. You can put the same toy twice!

Real-world: A dynamic shopping list where you can add items as you remember them.

Technical: ArrayList is a resizable array. It maintains insertion order and allows duplicates. Access is O(1), but removal can be O(n).

Program 8: Using Queue (PriorityQueue)

Demonstrates processing elements based on priority.

â˜• Java
import java.util.PriorityQueue;
import java.util.Queue;

public class PriorityQueueDemo {
    public static void main(String[] args) {
        Queue pq = new PriorityQueue<>();

        pq.add(30);
        pq.add(10);
        pq.add(20);

        // Head is the smallest element (natural ordering)
        System.out.println("Head: " + pq.peek()); // 10

        while (!pq.isEmpty()) {
            System.out.println("Processing: " + pq.poll());
        }
        // Output: 10, 20, 30
    }
}

Baby: A line where the smallest kid always goes first.

Real-world: Hospital emergency room triage (most critical first).

Technical: Unbounded priority queue based on a priority heap. Elements ordered by natural ordering or Comparator.

Program 9: Stack Implementation

Demonstrates LIFO (Last-In-First-Out) behavior.

â˜• Java
import java.util.Stack;

public class StackDemo {
    public static void main(String[] args) {
        Stack books = new Stack<>();

        books.push("Book 1");
        books.push("Book 2");
        books.push("Book 3");

        System.out.println("Top book: " + books.peek()); // Book 3

        while (!books.isEmpty()) {
            System.out.println("Popped: " + books.pop());
        }

    }
}

Technical: Extends Vector. LIFO structure. Synchronized (thread-safe) but often ArrayDeque is preferred for single-threaded use.

Program 10: Iterator Loop

Demonstrates safe removal of elements while iterating.

â˜• Java
import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;

public class IteratorDemo {
    public static void main(String[] args) {
        List names = new ArrayList<>();
        names.add("Alice");
        names.add("Bob");
        names.add("Charlie");

        Iterator it = names.iterator();
        while (it.hasNext()) {
            String name = it.next();
            if (name.equals("Bob")) {
                it.remove(); // Safe removal
            }
        }

        System.out.println("List after removal: " + names);
    }
}

Baby: A finger pointing to each item one by one.

Real-world: Checking items off a checklist.

Technical: Universal cursor. Allows safe removal during iteration (avoids ConcurrentModificationException).

Program 11: ListIterator Bidirectional

import java.util.ArrayList;
import java.util.List;
import java.util.ListIterator;

public class ListIteratorDemo {
    public static void main(String[] args) {
        List cities = new ArrayList<>();
        cities.add("New York");
        cities.add("London");
        cities.add("Tokyo");

        ListIterator it = cities.listIterator();

        System.out.println("Forward:");
        while (it.hasNext()) {
            System.out.println(it.next());
        }

        System.out.println("Backward:");
        while (it.hasPrevious()) {
            System.out.println(it.previous());
        }
    }
}

Baby: Walking forward and backward on a path.

Technical: Only for Lists. Can traverse in both directions and modify elements.

Program 13: Comparator Example

Demonstrates custom sorting logic.

â˜• Java
import java.util.ArrayList;
import java.util.Collections;
import java.util.Comparator;
import java.util.List;

public class ComparatorDemo {
    public static void main(String[] args) {
        List words = new ArrayList<>();
        words.add("Banana");
        words.add("Apple");
        words.add("Cherry");

        // Sort by length (custom logic)
        Collections.sort(words, new Comparator() {
            @Override
            public int compare(String s1, String s2) {
                return s1.length() - s2.length();
            }
        });

        System.out.println("Sorted by length: " + words);
    }
}

Baby: Sorting toys by color instead of size.

Real-world: Sorting students by height instead of grades.

Technical: Defines custom ordering logic external to the class. Useful when you can't modify the class or need multiple sorting ways.

Program 14: ConcurrentHashMap Demo

Demonstrates thread-safe map operations.

â˜• Java
import java.util.concurrent.ConcurrentHashMap;
import java.util.Map;

public class ConcurrentMapDemo {
    public static void main(String[] args) {
        Map map = new ConcurrentHashMap<>();

        map.put("A", 1);
        map.put("B", 2);

        // Safe to iterate and modify (unlike HashMap)
        for (String key : map.keySet()) {
            if (key.equals("A")) {
                map.put("C", 3); // No Exception!
            }
        }

        System.out.println("Map: " + map);
    }
}

Baby: Many people writing in a notebook at the same time without fighting.

Real-world: A shared google doc.

Technical: Thread-safe map. Uses segment locking (bucket-level) for high concurrency. Does not throw ConcurrentModificationException.

Program 15: Student Record Manager

A real-world application combining multiple collections.

â˜• Java
import java.util.*;

class Student {
    String id;
    String name;
    
    public Student(String id, String name) {
        this.id = id;
        this.name = name;
    }
    
    public String toString() { return name + " (" + id + ")"; }
}

public class StudentManager {
    public static void main(String[] args) {
        // Map to store students by ID (Fast lookup)
        Map studentMap = new HashMap<>();
        
        // List to maintain order of registration
        List registrationList = new ArrayList<>();
        
        // Set to ensure no duplicate IDs
        Set existingIds = new HashSet<>();
        
        // Add Student Logic
        addStudent("S01", "Alice", studentMap, registrationList, existingIds);
        addStudent("S02", "Bob", studentMap, registrationList, existingIds);
        addStudent("S01", "Alice Duplicate", studentMap, registrationList, existingIds); // Duplicate
        
        System.out.println("All Students (Map): " + studentMap);
        System.out.println("Registration Order: " + registrationList);
    }
    
    static void addStudent(String id, String name, Map map, 
                          List list, Set ids) {
        if (ids.contains(id)) {
            System.out.println("Error: Student ID " + id + " already exists!");
            return;
        }
        
        Student s = new Student(id, name);
        map.put(id, s);
        list.add(s);
        ids.add(id);
        System.out.println("Added: " + s);
    }
}

Baby: Managing a classroom.

Real-world: School database system.

Technical: Combines Map (lookup), List (history), and Set (uniqueness) to build a robust system.
